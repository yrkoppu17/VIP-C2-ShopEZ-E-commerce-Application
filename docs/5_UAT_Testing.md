# Phase 5: User Acceptance Testing (UAT)

This document contains the functional verification test scenarios conducted on the **shopEZ** platform to confirm that all operations comply with system requirements and product user stories.

---

## User Acceptance Testing (UAT) Ledger

| Test Case ID | Feature Scope | Steps to Execute | Expected Verification Result | Actual Outcome | Status |
| :--- | :--- | :--- | :--- | :--- | :--- |
| **UAT-001** | User Registration | 1. Navigate to Registration page.<br>2. Fill in valid Name, email address, and password.<br>3. Submit the registration form. | System creates a new profile in MongoDB, hashes the password, signs a session cookie, and loads the Home dashboard. | Profile added to MongoDB with Bcrypt hashed password; page loads user session immediately. | **Pass** |
| **UAT-002** | User Login & RBAC | 1. Navigate to Sign In page.<br>2. Input registered user credentials.<br>3. Submit the credentials.<br>4. Attempt to access `/admin` panel route directly. | Server grants signed JWT token stored in cookie. The user profile loads. DIRECT navigation to `/admin` must block and redirect standard users to homepage. | Credentials verified; JWTcookie set successfully; direct access to admin URL blocks standard customers. | **Pass** |
| **UAT-003** | Product Search & Filter | 1. Navigate to Catalog search.<br>2. Type query "Shirt".<br>3. Toggle price slider under "$50".<br>4. Select rating filter "4 Stars & Up". | Store catalog displays autocomplete suggestions. The search outputs matching items restricted under the $50 range holding ratings >= 4 stars. | Live suggestions filter items correctly. Price range and ratings sort listings instantly. | **Pass** |
| **UAT-004** | Cart Persistence & Wishlist Move | 1. Open Product details page.<br>2. Click "Add to Cart" and select quantity.<br>3. Go to Wishlist page.<br>4. Click "Move to Cart" for a favorited item.<br>5. Reload the browser page. | Cart counter increments on the navigation bar. Cart total updates. The wishlist item transfers to cart, clearing from wishlist. Reloading preserves the cart items. | Cart state is saved to React state context, synced to localStorage. Wishlist move transfers the item and clears favorite. | **Pass** |
| **UAT-005** | Checkout, Coupons & Stripe Payment | 1. Click Checkout button in Cart.<br>2. Pick a saved address.<br>3. Input active promotional coupon code.<br>4. Pick standard/express shipping speed.<br>5. Fill mock credit card inputs.<br>6. Click "Place Order". | Coupon applies corresponding discount. System calculates taxes (15% VAT) and shipping fees. Client sends intent request. Order details save, product stock decreases, and receipt renders. | Discounts compile. Intent returns secret. Product stocks decrement automatically in database. Order confirmation page loads. | **Pass** |
| **UAT-006** | Admin Catalog CRUD | 1. Log in as an Administrator.<br>2. Access Admin Dashboard panel.<br>3. Select Products catalog menu.<br>4. Click "Create Product" and fill details.<br>5. Select local image file to upload. | Image file is sent to Cloudinary API, returning CDN link. Form submit inserts product. Item displays instantly in public store catalog. | Image uploads to Cloudinary; new product is saved to MongoDB and displays on public pages. | **Pass** |
| **UAT-007** | Admin Status Overrides | 1. Log in as an Administrator.<br>2. Open Orders database panel list.<br>3. Change shipping status of a customer order to "Shipped".<br>4. Log back in as standard customer.<br>5. Review order receipt list page. | Admin dashboard updates state. Visual timeline tracker on the customer order receipt page updates stepper track from "Placed" to "Shipped". | Stepper track advances color. Database updates the order tracking timeline array dynamically. | **Pass** |
| **UAT-008** | Customer Product Reviews | 1. Log in as verified buyer.<br>2. Open product detail page.<br>3. Rate 5 stars and submit review text.<br>4. Attempt to write a second review.<br>5. Click "Helpful" upvote icon on another review. | System accepts rating, marks badge as "Verified Purchase", and recalculates average rating. Block subsequent review submissions. Upvote counter increments by 1. | System reviews database checks. Average score recalculates. Second submission fails validation check. Upvote tally increments. | **Pass** |
| **UAT-009** | Admin Customer Controls | 1. Log in as an Administrator.<br>2. Open Customer accounts manager panel.<br>3. Locate target user and toggle to "Blocked".<br>4. In a separate tab, attempt to log in using that blocked user account. | Account status updates to active=false. User is barred from logging in; system displays "Your account is deactivated" warning toast. | User is blocked. Login endpoint rejects request and throws error. | **Pass** |
